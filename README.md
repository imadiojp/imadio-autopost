# imadio autopost

X (Twitter) の投稿を事前にスケジュールして自動投稿できるWebアプリケーションです。

## 主な機能

- ✅ **投稿スケジューリング** - 好きな日時に投稿を予約
- ✅ **スレッド投稿** - 複数の連続ツイートを自動投稿
- ✅ **マルチアカウント対応** - 最大5つのXアカウントに同時投稿
- ✅ **画像添付** - 最大4枚の画像を投稿
- ✅ **カレンダービュー** - 週単位で投稿を視覚的に管理
- ✅ **自動リトライ** - 失敗した投稿を自動的に再試行
- ✅ **一括停止** - すべての予約投稿を一時停止
- ✅ **投稿履歴** - 過去の投稿を確認
- ✅ **レスポンシブデザイン** - スマホ・タブレット・PCに対応

## 使い方 (ユーザー向け)

### 1. アプリにアクセス
デプロイされたURLにアクセス

### 2. アカウント登録
メールアドレスとパスワードで登録

### 3. Xアカウント連携
「X連携」ページで「アカウントを追加」をクリックして、Xの認証画面で承認

### 4. 投稿を作成
- 「投稿作成」ページでテキストを入力
- 日時を設定
- 投稿先アカウントを選択
- 「投稿を予約」をクリック

### 5. 自動投稿
設定した時刻に自動的に投稿されます!

## デプロイ方法 (開発者向け)

詳細は [DEPLOYMENT.md](./DEPLOYMENT.md) を参照

**簡単な手順:**
1. X Developer Portalでアプリを作成
2. RailwayでバックエンドをデプロイGitHub連携)
3. Vercelでフロントエンドをデプロイ (GitHub連携)
4. 環境変数を設定

所要時間: 約15分

## 技術スタック

- **フロントエンド**: React 18 + TypeScript + Vite + Tailwind CSS
- **バックエンド**: Node.js + Express + SQLite + twitter-api-v2
- **デプロイ**: Vercel (フロントエンド) + Railway (バックエンド)

## ローカル開発

```bash
# フロントエンド
npm install
npm run dev

# バックエンド (別ターミナル)
cd server
npm install
cp .env.example .env
# .envを編集してX APIの認証情報を設定
npm run dev
```
